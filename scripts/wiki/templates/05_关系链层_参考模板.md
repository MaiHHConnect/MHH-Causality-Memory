# 关系链层书写规范

## 什么是关系链

关系链是把事件之间连起来的层。

- "这次争执 → 导致后来不联系"
- "某次合作 → 引出下一次机会"

**本质：记忆不是点，是网。**

## 命名格式

```
[主体A]-[主体B]_[关系类型].md
```

例如：
- `浩哥-牛马2号_主控执行关系.md`
- `小红红-塔姐_同属运营Agent关系.md`

## Frontmatter 格式

```yaml
---
title: 关系标题
type: relationship
from: [主体A]
to: [主体B]
relation: [关系类型：因果/引出/相关/合作/对抗/主控-执行]
summary: 这段关系的本质是什么
strength: [强/中/弱]
created: 2026-04-18
updated: 2026-04-18
---
```

## 正文内容

```markdown
## 关系定义

- **A**：
- **B**：

## 沟通模式

- 工具：
- 频率：
- 风格：

## 协作项目

| 项目 | A角色 | B角色 |
|------|--------|--------|

## 关键节点

- **日期**：事件描述

## 趋势

[关系在往哪个方向发展：变强？变弱？稳定？]
```

## 示例

### 示例：浩哥与牛马2号

```yaml
---
title: 浩哥与牛马2号_关系档案
type: relationship
from: 浩哥
to: 牛马2号
relation: 主控-执行
summary: 浩哥是决策者，牛马2号是执行者和协调者。沟通高效，浩哥给方向，牛马2号落地。
strength: 强
created: 2026-04-18
updated: 2026-04-18
---

## 关系定义

- **浩哥**：发指令、拍板决策、对结果负责
- **牛马2号**：接收指令、执行任务、协调子Agent、汇报结果

## 沟通模式

- 工具：钉钉（主要）、OpenClaw Web
- 风格：浩哥简洁直接，给结论不要方案B；牛马2号快速执行
- 频率：随时在线，有事直接发消息

## 协作项目

| 项目 | 牛马2号角色 |
|------|------------|
| 小红红 | 监控+协调 |
| 塔姐 | 监控+协调 |
| aidn(Codex) | 监控 |
| 记忆系统 | 主要维护者 |

## 关键节点

- **2026-04-18**：确立四层记忆结构，关系进一步深化为"系统共建"

## 趋势

关系稳定，协作默契度持续提升。
```

## 关系类型说明

| 类型 | 说明 |
|------|------|
| `因果` | A 直接导致 B |
| `引出` | A 间接引出 B（机会/合作） |
| `相关` | A 和 B 同主题/同上下文 |
| `对比` | A vs B |
| `合作` | A 和 B 共同完成某事 |
| `主控-执行` | A 决策，B 执行 |
| `同属` | A 和 B 属于同一系统/团队 |
