# 事件层书写规范

## 命名格式

```
YYYY-MM-DD_HHMM_[主体]_[动作]_[结果].md
```

例如：
- `2026-04-18_2136_浩哥重构记忆系统.md`
- `2026-04-18_2339_记忆关联模式确认.md`

## Frontmatter 格式

```yaml
---
title: 事件标题
type: event
date: 2026-04-18
time: 21:36 GMT+8
actors: [浩哥, 牛马2号]
location: 钉钉
action: 浩哥提出自然记忆四层结构理论，牛马2号实现wiki重构
result: 确立四层结构：事件层+时间线+关系链+抽象总结
emotions: [认同, 高效, 清晰]
tags: [记忆系统, 设计, wiki重构]
related_events: []
---

## 事件

[这里描述发生了什么：谁+在哪里+做了什么+结果]

## 内容摘要

[具体内容细节，分类记录]

## 结果

- [结果1]
- [结果2]

## 意义

[这件事意味着什么？对未来有什么影响？]
```

## 书写原则

**✅ 最小可记忆单元：**
```
谁 + 在哪里 + 做了什么 + 结果 + 情绪/标签
```

**✅ 必须能回答：**
> "这件事意味着什么？"

**✅ 每次写入时要想：**
> 未来这个事件能触发什么关联？

**❌ 不是流水账，格式示例：**

**错误**："今天浩哥让我整理记忆系统，我整理了"

**正确**：
```
谁：浩哥
在哪里：钉钉
做了什么：提出自然记忆四层结构理论
结果：确立四层结构（事件/时间线/关系链/抽象总结）
情绪：[认同，高效]
```

```yaml
浩哥在钉钉发来一套记忆结构理论，要求牛马2号按这个重构 wiki 和 Dream 系统。

> 存的不是数据，是一个人的经历结构。

### 四层结构
| 层级 | 名称 | 占比 |
|------|------|------|

| 第4层 | 抽象总结 | 15% |

## 结果

- [x] 创建四层目录结构
- [x] 重写 WIKI.md
- [ ] 事件层迁移
- [ ] Dream 系统对接

## 意义

这是 wiki 从"存档结构"转向"自然记忆结构"的起点。
```

## 存放位置

- `wiki/syntheses/events/by-date/` — 按日期归档
- `wiki/syntheses/events/by-project/` — 按项目归档
- `wiki/syntheses/events/by-person/` — 按人物归档

建议：**至少在 by-date 存一份**，by-project / by-person 按需建立。
